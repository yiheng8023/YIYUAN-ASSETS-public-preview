# Security Policy

Do not file public issues containing exploit details, secrets, credentials,
customer data, tenant data, private logs, or protected implementation details.

During public preview, security reports should be sent through a private,
controlled channel designated by the maintainers. If no private channel has
been announced yet, open a public issue only to request a private reporting
route and do not include sensitive details.

Public security coordination is allowed only for non-sensitive status,
documentation, or routing questions.
