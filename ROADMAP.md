# Roadmap

## Public Preview Surface

- Keep public entry material clear and bounded.
- Publish public-preview API and SDK surfaces only after boundary review.
- Add realistic examples that do not expose protected internals.
- Keep support, contribution, and security channels explicit.

## Semi-Open Expansion

- Add more public adapter or SDK surfaces when they are safe to disclose.
- Keep protected core authority and commercial orchestration private.
- Use public feedback as candidate signals, not automatic truth.
- Preserve a clean separation between public surface and private core.

## Deferred

- Production SLA.
- Final pricing and billing.
- Public Discussions at scale.
- GitHub Wiki as a parallel truth source.
- Full open-source licensing.
